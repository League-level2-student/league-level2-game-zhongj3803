Ball Bounce

by: zhongj3803

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I've seen games like Atari Breakout and I figured making something like a classic arcade game would be most suitable for this project. It has a level of complexity but isn't unbelievably hard like something such as Minecraft. Atari Breakout's game mechanics are still rather complicated with the presence of the blocks, so I have instead designed a game that has similar ball movement and rules, except the actual blocks of Atari are not present.

2. What I Learned

Based on the conversation with my teacher, I would say that for right now the game is a good start. However, for more complexity in the future, I could try and design the game so that there would be blocks, just like in the actual Atari Breakout game.

3. Architecture of My Game

My game has 5 classes. The Ball and Board classes coordinate movement of the ball and board, respectively, the GameObject class holds the variables and methods that the Ball and Board classes inherit, the GamePanel class helps to coordinate the interaction between the board and ball, and the BallGame class displays the panel via a JFrame. The Ball and Board classes mainly just had methods that moved each object, but it is actually the GamePanel class that helped to change the movement of these objects depending on collisions.

4. Technical Implementation

I'd say the use of inheritance was a good move on my part. I initially had not wanted to do it because inheritance was always something that I felt was kind of weird (especially the super and stuff), but I decided to do it just for a challenge. I had put the variables the Ball and Board class had in common in the GameObject class and set up a simple constructor. I didn't have any methods yet, but based on a suggestion from my teacher, and through my own consideration, I figured that the addition of an update method, such that it would update the collision box of the ball or board, would be helpful because both required the same exact code. I was always only used to classes inheriting variables, but from this I learned how to use inherited methods via the super "thing".

5. Most Challenging Part

I think the general challenge was always coming up with where to put different code and how to sort the classes. For example, I initially had wanted to check collisions in each of the Ball and Board classes, but then I came to a road block during that route, and based on my evaluation of the League Invaders project, I figured it would be more helpful to put all of that collision checking into a GamePanel class, which I did do later on.

6. Future Fixes and Updates

There is a tiny bug where if ball hits the side of the board, it can end up hitting the board multiple times, which can increase the score by more than 1 (which is not good). I have no clue how to fix this bug because it's just something that seems to be inherently part of the ball's movement, but if I were to do this sometime in the future, I'm hoping I could find a way to program smoother movement such that these funky bugs don't show up.
